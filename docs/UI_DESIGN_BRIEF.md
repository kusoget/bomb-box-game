# 爆弾箱ゲーム — UI 制作指示書（メインゲーム画面）

## 1. ゲーム概要

オンライン2人対戦の心理戦ゲーム。
12個の「箱」が円形に並ぶ盤面で、片方が爆弾を仕掛け、もう片方が箱を選んで開ける。
役割は毎ラウンド交代する。

### 役割
- **仕掛け人 (Switcher)**: 1つの箱に爆弾をセットする
- **解除役 (Sitter)**: 開ける箱を1つ選ぶ

### 1ラウンドの流れ
1. **setting_trap** — 仕掛け人が爆弾を箱にセット（解除役には見えない）
2. **selecting_chair** — 解除役が箱を選ぶ
3. **confirming** — 解除役が確定ボタンを押す
4. **revealing** — 結果発表（2秒間）
5. **round_end** — 次のラウンドへ（役割交代）

### 得点と勝敗
- セーフ（爆弾なし）→ 選んだ箱の番号（1〜12）が得点になる
- 爆発（爆弾あり）→ 自分の得点リセット + ライフ1つ消費（最初は3つ）
- 開けられた箱は盤面から消える（次以降に出現しない）
- **勝利条件**: ① 40点先取 / ② 相手のライフを0にする / ③ 残り1箱になった時点で得点が高い方の勝ち

## 4. UI要素の詳細仕様

### 4.1 スコアボード（画面上部）
左右にプレイヤー1とプレイヤー2を表示し、中央にフェーズ表示帯を置く。

各プレイヤーカードの内容:
- **アバター画像** (60×60, 円形クロップ。`user_a.png` / `user_b.png` を使用)
- **プレイヤー名**
- **得点** (大きな数字、2.2rem程度)
- **ライフ表示** (ハート3つ。消費したものは黒、残ってるものはピンク `#FF2D55` + 黒縁取り)
- **現在の手番のプレイヤーは強調表示** (アクティブ感を出す)

**フェーズ表示帯**（スコアの下、画面幅いっぱい）:
- 黄色背景 `#FFFF00`、太字、中央寄せ、約2rem
- 自分のアクションが必要なときは目立たせる（点滅 or 別色）
- 表示文言の例:
  - 爆弾セット中（自分） → 「爆弾をセットして！」
  - 爆弾セット中（相手） → 「相手が爆弾をセット中」
  - 箱選択中（自分） → 「箱からポイントをGET！」
  - 箱選択中（相手） → 「相手が箱を選択中」
  - 確定中（自分） → 「決定しますか？」
  - 結果発表 → 「結果発表」
  - ゲーム終了 → 「ゲーム終了」

### 4.2 メイン盤面（中央エリア）
**12個の箱が円形に配置**される。最大幅400pxの正方形コンテナ内に配置。

**箱（Chair）の見た目**:
- 円形 (70×70px)、白背景、黒3pxボーダー、黒ハードシャドウ (4px 4px 0 0 #000)
- 中央に番号（1〜12）を大きく表示 (2.2rem, 太字)
- ホバー時: 1.05倍にスケール
- **選択中**: 黄色背景 + 1.1倍スケール
- **爆弾セット中（仕掛け人視点のみ）**: 黄色背景 + 💣アイコン重ねる
- **開封済み**: 非表示 (visibility: hidden)
- **感電/爆発中**: 揺れアニメ + 赤フラッシュ

**中央エリア**:
- 通常は空
- 仕掛け人が爆弾セット中、解除役の画面では中央に大きな 💣 + 「相手が爆弾セット中」と表示

**手番の背景色**:
- プレイヤー1のターン → メインエリア背景 紫 `#9b59b6`
- プレイヤー2のターン → メインエリア背景 オレンジ（CSS変数 `--pop-orange`、未定義なら定義する）
- 切替えは 0.5s ease トランジション

### 4.3 アクションボタン（フッター固定）
- 画面下中央、ピンク `#FF2D55` 背景の角丸ピル型ボタン（最大320px）
- 1.8rem、太字、白文字
- **爆弾セットフェーズ**: 「爆弾をセットする」（赤背景 `#d63031`）
- **箱選択フェーズ**: 「この箱にする」（ピンク背景）
- 選択していない間は disabled（グレー）

### 4.4 チャット（サイドバー / モバイルはフローティング）
- デスクトップ: 右側320px固定、左に黒3pxボーダー、白背景
- メッセージ一覧 + 入力欄
- スマホ: 右下に💬ボタン、タップで全画面オーバーレイ

### 4.5 オーバーレイ系
- **ルーレット（初回ラウンド開始時）**: 「誰が先か」を抽選するアニメ、プレイヤー名を交互にハイライト → 決定時にズーム表示
- **結果オーバーレイ**: revealingフェーズで「セーフ！+N点」or「爆発！」を画面中央に2秒間表示
- **ゲームオーバーオーバーレイ**: 勝者名 + 「もう一度遊ぶ」「ホームへ」ボタン
- **トースト通知**: 画面上部から滑り込み、2〜3秒で消える（例: 「爆弾をセットしました！」）
- **感電エフェクト（ElectricEffect）**: 爆発時に画面全体に稲妻フラッシュ

## 5. デザインシステム（Retro Pop テーマ）

### 5.1 カラーパレット
| 用途 | 値 |
|---|---|
| 背景（メイン） | `#5EEAD4` ミントグリーン |
| 背景（カード） | `#FFFFFF` 白 |
| アクセント1（ピンク） | `#FF2D55` |
| アクセント2（黄色） | `#FFFF00` |
| アクセント3（紫＝P1ターン） | `#9b59b6` |
| アクセント4（オレンジ＝P2ターン） | 要定義（例: `#FF8C42`） |
| アクセント5（赤＝危険） | `#d63031` |
| テキスト | `#000000` ピュアブラック |

### 5.2 スタイルの特徴
- **太い黒ボーダー**（3px solid #000）が基本
- **ハードシャドウ**（`4px 4px 0px 0px #000`、ぼかしなし）
- **角丸ピル型ボタン**（border-radius: 999px）
- **ポルカドット背景**（body に `radial-gradient` で水玉模様）
- フラットなレトロポップ感、ぼかしやグラデーションは使わない

### 5.3 タイポグラフィ
- 日本語: `Dela Gothic One`（next/fontで読み込み済み）
- 数字・英字: `Fredoka` 900ウェイト
- ボタン・数字は特に大きめに

### 5.4 アニメーション
- トランジション基準: `0.15s` (fast) / `0.3s` (normal) / `0.4s cubic-bezier(0.175, 0.885, 0.32, 1.275)` (bouncy)
- 用意したいキーフレーム: `popIn`（拡大フェードイン）、`bounce`（上下）、`pulse`（透明度往復）

## 7. データと型（参考: 触らなくてよい）

```ts
type GamePhase =
  | 'waiting' | 'setting_trap' | 'selecting_chair'
  | 'confirming' | 'revealing' | 'round_end' | 'game_over';

interface Chair { id: number; isRemoved: boolean; isTrapped: boolean; }

interface Player {
  id: string; name: string; avatar: string;
  playerNumber: 1 | 2; score: number; shockCount: number;
}

interface GameState {
  chairs: Chair[];
  player1Score: number; player2Score: number;
  player1Shocks: number; player2Shocks: number;
  currentRound: number;
  phase: GamePhase;
  currentSitterId: string | null;
  currentSwitcherId: string | null;
  trappedChair: number | null;   // 仕掛け人にのみ見える
  selectedChair: number | null;
  winner: string | null;
}
```

## 8. 制約・お願い
- **ロジックは変更しない**（GameBoardのprops契約、useEffectの分岐条件は維持）
- 既存ファイルの**CSS Modules書き換え + JSXの構造調整**が中心
- レスポンシブ必須（PC / スマホ縦持ち両対応）
- 既存の `globals.css` の CSS変数はそのまま活用可
- 命名は現状の "Chair" のままでOK（旧テーマの名残だが、UI上は「箱」と表示）

## 9. 最重要ポイント（優先順）
1. 12個の箱の **円形レイアウトの美しさ**（中央に演出スペースを残す）
2. **手番が誰なのか一目でわかる**こと（背景色 / フェーズ帯 / アクティブ強調）
3. 結果発表時の **爆発演出のインパクト**
4. **モバイル片手プレイの操作性**（ボタン位置、タップしやすいサイズ）
